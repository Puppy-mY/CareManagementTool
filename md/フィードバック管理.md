## 新機能: フィードバックシステムの設計

### 概要
ユーザーから開発者への要望・不具合報告を収集し、双方向のコミュニケーションを可能にするシステム

### 機能要件

#### フェーズ1: 基本機能（今回実装）
1. **ヘッダーボタン**
   - 「開発者への要望」ボタンをヘッダーに配置
   - モーダルでフォームを表示

2. **フィードバックフォーム**
   - カテゴリ選択（不具合/新機能提案/改善要望/その他）
   - タイトル入力
   - 詳細内容の記入欄（テキストエリア）
   - 優先度選択（高/中/低）
   - 現在のページURL自動取得
   - 送信ボタン

3. **データベース保存**
   - Feedbackモデルの作成
   - ユーザー情報、カテゴリ、内容、ステータスを保存

4. **開発者用管理ページ**
   - フィードバック一覧表示
   - 詳細表示・返信機能
   - ステータス管理（未対応/対応中/完了/却下）

#### フェーズ2: コミュニケーション機能
1. **返信機能**
   - 開発者からユーザーへの返信
   - スレッド形式でのやり取り
   - 通知機能（新着返信）

2. **通知システム**
   - 新規フィードバック → 開発者へ通知
   - 返信 → ユーザーへ通知
   - メール/システム内通知

#### フェーズ3: 高度な機能
1. **投票機能**
   - 他のユーザーも要望に賛成できる
   - 投票数で優先度を判断
   - 人気の要望一覧

2. **添付ファイル**
   - スクリーンショット添付
   - ファイルアップロード

3. **ユーザー向けフィードバック一覧**
   - 自分が投稿したフィードバック
   - 対応状況の確認

### データベース設計

#### Feedbackモデル
```python
- id: 主キー
- user: ユーザー（外部キー）
- category: カテゴリ（不具合/新機能/改善/その他）
- title: タイトル
- content: 詳細内容
- priority: 優先度（高/中/低）
- status: ステータス（未対応/対応中/完了/却下）
- page_url: 報告元のページURL
- created_at: 作成日時
- updated_at: 更新日時
```

#### FeedbackReplyモデル（フェーズ2）
```python
- id: 主キー
- feedback: フィードバック（外部キー）
- user: 返信者（外部キー）
- content: 返信内容
- created_at: 作成日時
```

#### FeedbackVoteモデル（フェーズ3）
```python
- id: 主キー
- feedback: フィードバック（外部キー）
- user: 投票ユーザー（外部キー）
- created_at: 投票日時
```

## バグ修正: ステータス更新エラーの解決

### 31. ステータス更新時の400エラーを修正（完了）
- **日時**: 2025-11-18
- **問題**: ステータスバッジをクリックして変更しようとすると「ステータスの更新に失敗しました: 不明なエラー」が表示される
- **原因**:
  - JavaScriptがJSONデータとして送信（`Content-Type: application/json`）
  - サーバー側が `request.POST.get('status')` でフォームデータとして受け取ろうとしていた
  - データ形式の不一致により400エラーが発生
- **修正内容**:

#### 1. jsonモジュールのインポート追加（clients/views.py）
```python
import json
```

#### 2. feedback_status_update関数の修正（clients/views.py）
- `request.POST.get('status')` から `json.loads(request.body)` に変更
- JSONデータを正しくパースして受け取るように修正
- エラーハンドリングを追加:
  - `JSONDecodeError`: JSONのパース失敗時
  - `Exception`: その他のエラー時
- より詳細なエラーメッセージを返すように改善

#### 修正後のコード
```python
@login_required
def feedback_status_update(request, pk):
    """フィードバックステータス更新（AJAX）"""
    # 管理者権限チェック
    if not request.user.is_staff and not request.user.is_superuser:
        return JsonResponse({'success': False, 'message': '権限がありません'}, status=403)

    if request.method == 'POST':
        try:
            feedback = get_object_or_404(Feedback, pk=pk)

            # JSONデータを読み取る
            data = json.loads(request.body)
            new_status = data.get('status')

            if new_status in dict(Feedback.STATUS_CHOICES):
                feedback.status = new_status
                feedback.save()
                return JsonResponse({
                    'success': True,
                    'message': 'ステータスを更新しました',
                    'status': feedback.get_status_display()
                })
            else:
                return JsonResponse({'success': False, 'message': '無効なステータスです'}, status=400)
        except json.JSONDecodeError:
            return JsonResponse({'success': False, 'message': 'JSONのパースに失敗しました'}, status=400)
        except Exception as e:
            return JsonResponse({'success': False, 'message': f'エラーが発生しました: {str(e)}'}, status=500)

    return JsonResponse({'success': False, 'message': '無効なリクエストです'}, status=400)
```

```python
        return redirect('my_feedback_list')
```

```python
def check_inactive_users(sender, request, user, **kwargs):
    """ログイン時に1ヶ月以上ログインがないユーザーを自動無効化"""
    one_month_ago = timezone.now() - timedelta(days=30)

    inactive_users = User.objects.filter(
        last_login__lt=one_month_ago,
        profile__is_active=True
    ).exclude(pk=user.pk)

        profile = inactive_user.profile
        profile.is_active = False
        profile.save()
```

**動作仕様**:

**実装方法**:

所属部署は`DEPARTMENT_CHOICES`で定義されており、データベースには英語コード（`kyotaku`、`helper`等）が保存されていますが、ユーザーは日本語（居宅、ヘルパー等）で検索することを想定しています。

**検索ロジック** (`clients/views.py`):
```python
if search_query:
    # 所属部署の日本語ラベルから英語コードを検索
    department_codes = [code for code, label in UserProfile.DEPARTMENT_CHOICES
                       if search_query in label or search_query in code]

    # Q オブジェクトを構築
    query = (
        Q(username__icontains=search_query) |
        Q(profile__last_name__icontains=search_query) |
        Q(profile__first_name__icontains=search_query) |
        Q(profile__last_name_kana__icontains=search_query) |
        Q(profile__first_name_kana__icontains=search_query)
    )

    # 所属部署に該当するコードがあれば追加
    if department_codes:
        query |= Q(profile__department__in=department_codes)

    users = users.filter(query)
```

**動作仕様**:
1. ユーザーが検索ワードを入力（例: 「ヘルパー」）
2. `DEPARTMENT_CHOICES`から該当する日本語ラベルまたは英語コードを検索
3. 該当する英語コードのリストを取得（例: `['helper', 'helper_general_affairs']`）
4. そのコードを持つユーザーを検索結果に含める
