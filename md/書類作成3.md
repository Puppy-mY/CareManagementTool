### マスタデータ編集ドロワーの設計方針

#### 画面構成（アコーディオン形式）
書類作成画面では、利用者に関する情報を3つのアコーディオンに分けて表示：

1. **【利用者基本情報】**（アコーディオン）
   - 氏名、フリガナ、生年月日、住所、被保険者番号
   - すべてreadonly
   - [利用者マスタデータを編集]ボタン

2. **【医療保険情報】**（アコーディオン）
   - 加入保険、保険者番号、被保険者番号、記号番号
   - すべてreadonly
   - [利用者マスタデータを編集]ボタン

3. **【主治医情報】**（アコーディオン）
   - 医療機関名、住所、電話番号、主治医氏名
   - すべてreadonly
   - [利用者マスタデータを編集]ボタン

#### ドロワー統合設計
**重要**: 上記3つのアコーディオンのどの[編集]ボタンを押しても、**同じ利用者マスタ編集ドロワーが開く**。

ドロワー内のレイアウト：

│ │ • 保険者番号                             │     │
│ │ • 被保険者番号                           │     │
│ │ • 記号                                   │     │
│ │ • 番号                                   │     │
│ └──────────────────────────────────┘     │

**メリット**:
- 画面上は見やすく分かれている（3つのアコーディオン）
- 編集は1箇所でまとめてできる（1つのドロワー）
- マスタデータの管理単位が明確（利用者という単位で一元管理）
- 将来的な拡張がしやすい（利用者に関する新しい情報を追加しやすい）

#### マスタデータの分類
- **利用者に関すること**: 基本情報 + 医療保険情報 + 主治医情報 → 1つのドロワーで一元管理
- **地域包括支援センターに関すること**: 事業所番号、名称、住所、電話番号、担当者名 → 地域包括マスタ編集ドロワー
- **居宅介護支援事業所に関すること**: 事業所番号、名称、住所、電話番号、担当者名 → 居宅介護マスタ編集ドロワー

### データの自動反映方針

#### マスタデータからの自動反映
以下の情報はマスタデータから自動反映される（ほぼ変わらないデータ）：
- **利用者基本情報**: 氏名、フリガナ、生年月日、性別、住所、郵便番号、電話番号、被保険者番号など（`Client`モデル）
- **認定調査場所（安濃津ろまん選択時）**: 施設名、住所、郵便番号、電話番号など（新規マスタまたは固定値）

#### アセスメントDBからの自動反映（編集可能）
以下の情報はアセスメントシートのDBから取得して自動反映するが、**都度変わる可能性があるため編集可能**とする：
- **医療保険情報**: 加入保険種別、保険者番号、被保険者番号、記号番号など
- **主治医情報**: 医療機関名、住所、電話番号、主治医氏名など

**理由**: 医療保険は後期高齢者への移行などで変わる可能性があり、主治医も変更される可能性があるため、マスタではなく「その都度の入力データ」として扱う。

#### 将来的な拡張
よく使う主治医情報などは、将来的にマスタ化（選択肢として保持）することも検討。

### 認定調査場所の扱い

#### プルダウン選択肢
1. **サービス付き高齢者向け住宅　安濃津ろまん**
   - 選択時：施設の住所、郵便番号、電話番号を**自動反映**（固定値から）
   - 編集：不可（readonly）

2. **それ以外の場所**
   - 選択時：すべての項目を**手動入力**
   - 編集：可能

この方式により、頻繁に使う「安濃津ろまん」の場合は入力の手間を省き、その他の場所の場合は柔軟に対応できる。

#### 実装方式：固定値（JavaScript）

**方針**: シンプルさを優先し、固定値として実装。将来的に複数の施設を管理する必要が出た場合はモデル化を検討。

```javascript
// 認定調査場所の固定値
const INVESTIGATION_LOCATIONS = {
    'annou_roman': {
        name: 'サービス付き高齢者向け住宅　安濃津ろまん',
        postal_code: '514-0009',
        address: '三重県津市羽所町○○',
        phone: '059-123-4567'
    }
};

// プルダウン変更時の処理
function onInvestigationLocationChange() {
    const selectedValue = document.querySelector('[name="investigation_location"]').value;

    if (selectedValue === 'annou_roman') {
        // 安濃津ろまんの情報を自動反映 + readonly
        const data = INVESTIGATION_LOCATIONS.annou_roman;
        document.getElementById('facility_name').value = data.name;
        document.getElementById('facility_postal_code').value = data.postal_code;
        document.getElementById('facility_address').value = data.address;
        document.getElementById('facility_phone').value = data.phone;
        setFacilityFieldsReadonly(true);
    } else if (selectedValue === 'other') {
        // クリア + 編集可能
        clearFacilityFields();
        setFacilityFieldsReadonly(false);
    }
}
```

**メリット**:
- シンプルで実装が速い
- データベース不要、マイグレーション不要
- 変更が容易（JavaScriptを編集するだけ）

**将来的な拡張**:
- 複数の施設を管理する必要が出た場合は、InvestigationFacilityモデルを作成して管理画面から編集可能にする
