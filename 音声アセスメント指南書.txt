Claude Code 開発指南書（完成版）
Whisper疑似リアルタイム × GPTでアセスメントを自動仮入力する
0. ゴール定義
完成状態（MVP→拡張まで）

アセスメントシート画面に 「AI聞き取り開始」ボタン

押すと録音（例：5秒）→サーバ送信→Whisper文字起こし

GPTが 既存フォームの項目構造に合わせたJSON を返す

画面の該当項目に 仮入力（色付き） で反映される

さらに「次に聞くべき質問」を提案する（拡張タスク）

絶対ルール（医療・介護業務向け）

AIは確定しない（仮入力）

選択式（例：動作レベル）は 自動選択せず“候補提示”

すべてのAI入力は 編集・確定可能 にする

2. コア設計（Claude Codeに最初に渡す指示テンプレ）

以下を Claude Code に最初に貼ります（毎回使い回し）。

あなたはDjangoで構築された介護アセスメントシステムのAI機能追加を支援する開発アシスタントです。

前提：
- 手入力アセスメントは完成済み
- 今回は音声聞き取り支援を追加
- AI入力は仮入力扱い（自動確定しない）
- 段階的に実装する（小さく動かす→拡張）

必須要件：
- 食事アセスメントは以下の構造を持つ
  action_level（選択：自立/見守り/一部介助/全介助）※候補提示のみ
  meal_type（短文）
  tools（短文）
  detail（詳細文章）
- GPTの出力は必ずJSON（説明文禁止）
- Django APIとフロントJSを分離し、既存コードを壊さない

3. 実装フェーズ1（MVP）：音声→文字→項目JSON→画面反映

完了条件

Claude Codeへの指示例

Djangoに /api/ai/transcribe/ を追加してください。
入力: multipart/form-data の audio
出力: JSON {text: "..."}
内部: OpenAI whisper-1 を使用、language=ja
音声ファイルは保存しない

Task 3-2: GPT分類API（Django）を作る（テキスト入力で動かす）

やること

POST /api/ai/meal/extract/ を追加（まず食事だけ）

入力：text（Whisper結果）

出力：食事フォーム構造に合わせたJSON

GPT出力の正解フォーマット

{
  "meal": {
    "action_level_candidate": "自立",
    "meal_type": "刻み食",
    "tools": "スプーン",
    "detail": "自立して食事摂取可能だが、最近むせが増えているため刻み食。スプーン使用。",
    "confidence": 0.92
  }
}


※ action_level は確定させないので action_level_candidate にするのがおすすめ

完了条件

テキストを投げて、常にJSONで返ってくる（説明文ゼロ）

JSONのキーが毎回同じで、フロント実装が安定する

Claude Codeへの指示例

食事の抽出API /api/ai/meal/extract/ を作ってください。
入力: JSON {text: "..."}
出力: 必ず以下のJSON形式のみ
{
 "meal": {
  "action_level_candidate": "",
  "meal_type": "",
  "tools": "",
  "detail": "",
  "confidence": 0.0
 }
}
action_level_candidate は 自立/見守り/一部介助/全介助 のいずれか1つ。
わからない場合は空文字とし confidence を下げる。

7. 仕様の核心：項目構造を事前に伝えるべきか？

答え：必須です。
業務システムでは GPT に「作文」をさせると失敗します。
必ず フォームの型（項目名・制約・禁止事項） をプロンプトに書いて、「データ生成」に寄せます。

食事項目（プロンプトに必ず書く制約）

action_level_candidate は 4択のどれか、無理なら空

meal_type は形態だけ（常食/刻み/ミキサーなど）

tools は用具だけ（箸/スプーン/自助具など）

detail は記録文（簡潔、敬体不要、事実＋課題）

8. Claude Code運用ルール（重要）
ルールA：一度に1タスクだけ依頼する

「Task 3-2 だけ作って」

「そのテスト方法も書いて」
のように小さく

ルールB：必ず“完了条件”を提示して依頼する

「JSONが崩れないこと」

「説明文が混ざらないこと」

ルールC：プロンプトはファイル化

直書き禁止（保守不能になる）
