# Claude Code 開発指南書（完成版 v2 / iPhone対応）

Whisper疑似リアルタイム × GPTでアセスメントを自動仮入力する  
（Django既存アセスメントシステム拡張）

---

## はじめに（重要）

この文書は  
「すでにDjangoで構築済みの介護アセスメントシステム」に  
音声 × Whisper × GPT を後付けするための開発指南書である。

最優先事項：

- 現場で止まらないこと
- AIは補助であり決定しない
- すべて人間が確認・修正できる
- 技術より運用を優先する

---

# 0. ゴール定義

完成状態（MVP→拡張まで）

### 現場（iPhone）

- 利用者選択
- 「AI聞き取り開始」ボタン
- iPhoneで録音（30秒chunk）
- 数十秒遅れでアセスメント項目が仮入力される

### Web（既存Django）

- 仮入力確認
- 修正
- 確定
- 履歴管理

---

基本フロー：

iPhone録音  
→ 30秒単位で音声送信  
→ Whisper文字起こし  
→ GPTで構造化JSON生成  
→ Djangoで差分マージ  
→ UIに仮入力反映  

---

画面仕様：

- アセスメントシートに「AI聞き取り開始」
- AI入力は色付き仮入力
- action_levelは候補提示のみ
- 「AI入力解除」ボタンあり

さらに「次に聞くべき質問」を提案（拡張）

---

## 絶対ルール（医療・介護）

- AIは確定しない（仮入力のみ）
- 選択式は自動選択しない（候補提示）
- すべて編集可能
- 音声は保存しない（処理後即破棄）

# 2. Claude Code 初期指示テンプレ

あなたは Django で構築された介護アセスメントシステムに  
音声AI機能を追加する実装アシスタントです。

前提：

- 手入力アセスメントは完成済み
- iPhoneで録音し、サーバでWhisper処理
- AI入力は仮入力のみ
- 段階実装

必須：

- GPT出力は必ずJSON
- 既存コードを壊さない
- APIとフロントは分離
- プロンプトはファイル化

---

# 3. 実装フェーズ1（MVP）

目的：

「話す → 食事項目が入る」

---

## Task 3-1 Whisper文字起こしAPI

POST /api/ai/transcribe/

input: audio  
output:

{ "text": "..." }

音声ファイルは保存しない。

---

## Task 3-2 食事項目抽出API

POST /api/ai/meal/extract/

input:

{ "text": "..." }

output（固定）：

{
 "meal": {
  "action_level_candidate": "",
  "meal_type": "",
  "tools": "",
  "detail": "",
  "confidence": 0.0
 }
}

制約：

- action_level_candidate：
  自立 / 見守り / 一部介助 / 全介助
- 不明なら空文字
- 説明文禁止

---

## Task 3-3 プロンプト分離

prompts/meal_extraction.py

views.py直書き禁止。

---

## Task 3-4 iPhone録音 → 送信

- AVAudioRecorder
- 30秒chunk
- 連続送信

---

## Task 3-5 仮入力反映

- meal_type/tools/detail 自動入力
- action_level 候補表示のみ
- ai-draft クラス付与

---
