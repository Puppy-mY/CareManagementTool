# Googleスプレッドシート連携仕様（将来実装用メモ）

## 概要
現在Excel形式で出力しているアセスメントシートについて、Google Sheets APIを用いたスプレッドシートへの直接出力・連携を行うための要件および仕様案。

## 仕様（Googleスプレッドシートのファイル構成）
利用者の過去のアセスメントを見やすく管理するため、以下のデータ構造とする。

### 1. 1つの利用者につき1つのファイル（スプレッドシート）
- ある利用者（例：Aさん）のアセスメントを初めてスプレッドシートに出力した際、Googleドライブ上に「Aさんのアセスメント」というスプレッドシートを**新規作成**する。
- システム側で、どの利用者がどのスプレッドシート（Spreadsheet ID）に紐づいているかをデータベース等に保持する。

### 2. シート（タブ）はアセスメント単位
- 作成したスプレッドシートの中には、アセスメントごとにシート（タブ）を作成する。
- 1回目のアセスメントを作成して出力した場合は、「アセスメント①」（またはアセスメント作成日ごとの名前など）というシートが新しく作られる。

### 3. 同一アセスメントの再出力（上書き）
- すでに出力済みの「アセスメント①」の内容をシステム上で修正し、再度出力ボタンを押した場合。
- 新しいシートやファイルを作るのではなく、対象の利用者のスプレッドシートを開き、「アセスメント①」のシートを探して、その**中身（各セル）を最新の状態に上書き**する。

### 4. 新規アセスメントの出力（新規シートの追加）
- 新しく「アセスメント②」を作成し、出力ボタンを押した場合。
- 同じ利用者のスプレッドシートを開き、現在のシート群の右側などに「アセスメント②」という**新しいシート（タブ）を追加**してデータを書き込む。

---

## 期待される効果
- 何度出力・修正を行なってもファイルや不要なシートが乱立しないため、Googleドライブ内が整理された状態を保てる。
- 過去のアセスメントから最新のアセスメントまで、ひとりの利用者の履歴を1つのファイルのタブ切り替えだけで閲覧できる。
